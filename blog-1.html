A beginners Intro to Git and GitHub

Complete Guide on how to install Git and upload Projects to your GitHub Account

  • 06 Jun 2020

  • Coding

why Github? Well, It is primarily used for source code management in project development, but it can be used to keep track of changes in any set of files it is open source and free to use.
You can upload your project source code on github and access it remotely from any where. It's up to you that either you want keep your files Private or you can keep it as Public so anyone can get your souce files and get help

Git is a Distributed Version Control tool that is used to store different versions of a file in a remote or local repository. It is used to track changes in the source code. It allows multiple developers to work together.

    Cloning a repository

    When you create a repository on GitHub, it exists as a remote repository (i.e It will remain Online on your Github account website). But if you want to make changes in your file,then you may need the files to be downloaded on your system or laptop. Here comes the concept of Cloning.
    The process of copying the repository from GitHub to your local machine is called cloning. You can clone your repository to create a local copy on your computer and sync between the two locations.

    When you're working with a repository on GitHub, you might want to have it on your local machine to make it easier to work with. For example, to allow you to use your favorite editor, to add and remove files, or to fix conflicts. Cloning a repository pulls down a full copy of all the repository data that GitHub has at that point in time
    You can either clone your existing repository or, to contribute to a project, clone another person's existing repository..

    For this tutorial, we’ll be using the Spoon-Knife project , a test repository that’s hosted on GitHub.com that lets you test the Clone workflow.
