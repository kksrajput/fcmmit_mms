A full-featured e-mailing system for PHP projects

  • 06 Jun 2020
  • Rahul Kumar, 1NC17CS074 CSE,NCET
  • Coding

-> As a web developer, you need to make a project (web application) which interact with the client and administration on a quick basis. These things make you a professional web application management controller.
->Many aspects comes in the making of a professional platform to interact with the client user and admin. One of the many things is emailing the information. Either it is email verification on “signup pages”, or sending details of “contact us” directly to admin, or promotional emails to different platforms etc.
->Now I am going to reflect my knowledge about the emailing system which can be used in any project.
->As a amature web developer, everyone will start making their projects on localhost for testing and modification purpose. Then they deploy their work (project) globally. I am going to tell you about email system for both circumstances, localhost and globally-deployed.
->First I will tell you different ways to do so on the localhost. Then we will move upon a live server.

1: On localhost server



Prerequisite

  • Localhost server :- xampp or wampp (recommended:-xampp)
  • google business account

    • If you don’t have a local server then download one.
    If you dont have a google business account then you can make one by this option.

    Open the google account and go to “security” section.
    Turn OFF "two step verification"



    And turn ON the "less secure app access". So that localhost can access your google account to send messages.

    1st method on localhost

    using swiftmailer library

    The first method we are doing by using some library. For that you need "composer" in your system.
    Composer is a tool for dependency management in PHP. It allows you to declare the libraries on which your project depends and it will manage (install/update) them for you.

    If you want to check whether composer is there in your system or not then type “composer” in command prompt and press enter. If it will show “composer is not recognized as an internal or external command” then it means your system don’t have a "composer". You can download composer from here.

    After installing composer, type composer in command prompt. It will show you like below mentioned version, with many information alongwith that.


    Now you have to add the "swiftmailer" library into project library. For doing so, open command prompt. Go to your library by “cd” command, as example: 

    cd C:\xampp\htdocs\mailproject

    We have to use swiftmailer library for email purpose. you can download the library by executing this command on your command prompt 
    composer require "swiftmailer/swiftmailer:^6.0"
    Now go to your project and open folder, you should see these files

    Now create a php file named anything (example.php) in same project. And open that file with an editor.
    at starting of file, you need to add path of the autoload.php which reside in vendor folder, by using this code 
    require_once './vendor/autoload.php';
    Next thing you have to do is to create a object (transport) for swift smtp transport class and set the username and password
    Username and password should be the gmail id which you are using to send mail
    port number should be 465 and smtp_ssl should be ssl 
    $transport = (new Swift_SmtpTransport('smtp.gmail.com', 465, 'ssl'))
                        ->setUsername('sender@gmail.com')
                        ->setPassword('your password')
                        ;
    Now create a mailer variable by using the transport ($transport) which has been already created. 
    $mailer = new Swift_Mailer($transport);
    Now create a message which include receiver email address, subject, body, etc 
    $message = (new Swift_Message('Subject to send'))
                        ->setFrom('sender@gmail.com')
                        ->setTo('receiver@gmail.com')
                        ->setBody('message which you want to send ')
                        ;
    Now send the mail. 
    $mailer->send($message);
    The whole file should look like this: Now go to browser and execute this file.
    You will receive a mail on the reciever@gmail.com from sender@gmail.com.


    If you getting any doubt or problem then feel free to contact me. just drop a mail here.

    [NOTE] : you can take all the values from user and store in a php variable to use it in mail function.

    2nd method on localhost

    By editing ini files.

    -> Go to xampp folder which reside in your C drive.
    -> There you will find a folder naming “sendmail”, open it.
    -> In that folder you will find a file named “sendmail.ini”, now you have to open that file with any editor.
    -> Now go to line 14, there you will find smtp_server, uncomment it (if it is commented) by removing semicolon from the beginning.
    -> If you have to use google account for your mailing system, then edit

    smtp_server=smtp.gmail.com
    if yahoo 
    smtp_server=smtp.mail.yahoo.com
    now go to line 18, uncomment the line smtp_port. edit that line as 
    smtp_port=465                     //because both server run on port 465
    Go to line 26, uncomment the line smtp_ssl. Edit 
    smtp_ssl=ssl
    Then go to line 46, uncomment the line auth_username and auth_password, and edit as 
    auth_username=sender@gmail.com              //account which you created to send email to others
    auth_passsword=sender password
    Just save this file.

    Now go to your xampp folder, you will find a folder named “php”, open that, now u will find a file named “php.ini” , open that file with an editor.

    In that folder search for the “sendmail” or you can just find it on the 1049th line.

    Now one thing you should know that, the localhost mailfunction doesn’t allow you to set the authentication values as default. So you need to comment the line SMTP=localhost as 
    ;SMTP=localhost
    And smtp_port=25 as 
    ;smtp_port=25
    Now go to line 1050, there you will find sendmail_from , uncomment it and keep your email id as the same you kept before in sendmail.ini 
    sendmail_from=sender@gmail.com
    then go to line 1054, there you will find sendmail_path , edit that as the actual path of your sendmail function 
    sendmail_path = "C:\xampp\sendmail\sendmail.exe -t"
    Save it. Now you should restart your apache server so that your new php.ini should take place. (stop and start)


    almost everything is done. Now go to your htdocs (root) folder, choose any project and create a php file.
    Edit the code as Now go to browser and execute this file.
    You will receive a mail on the reciever@gmail.com from sender@gmail.com.

    If you getting any doubt or problem then feel free to contact me. just drop a mail here.

    [NOTE] : you can take all the values from user and store in a php variable to use it in mail function.

    On live server

    On live server, you don’t need compulsory to have google/yahoo account.
    You can create a mailing account from cPanel.
    When you buy a web server, the service provider provides a cPanel to manage your websites and settings.
    This is what a cPanel looks like which is provided by web service provider.


    To create a mailing account, go to Email Accounts under the EMAIL


    Click on “create” option


    Now choose the domain. If you have only one domain then it will be taken as default.
    Then choose a username. (this will be your user id. sender@domain.com)
    Enter a password Then click create.


    You have successfully created your mail account.
    Now go to any of yours project which is in the public_html
    And create a php file. Open in an editor and write these codes.
    Now open that file in browser
    You will receive email on receiver@gmail.com in few moments (2-3 minutes).
